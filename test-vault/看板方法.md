# 看板方法

看板（Kanban）是一种可视化工作管理方法，源自丰田生产系统。

## 核心实践

1. 可视化工作流
2. 限制在制品（WIP）
3. 管理流动
4. 明确流程策略
5. 建立反馈循环
6. 协作改进
7. 

## 与 Scrum 的区别

维度看板Scrum迭代持续流动固定冲刺角色无固定角色SM/PO/Team变更随时可加冲刺内不变

两者都属于 \[\[敏捷开发\]\] 的实践方式。

看板适合需求变化频繁、工作项类型多样或需要持续交付的团队，可通过累计流图、周期时间和吞吐量等指标持续优化效率。实际使用时，团队应从现有流程出发逐步改进，而不是一次性重构全部工作方式。

## 适用场景

- 运维团队
- 支持团队
- 持续交付的 \[\[项目管理\]\] 场景
