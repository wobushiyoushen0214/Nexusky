# Scrum 框架

Scrum 是 [[敏捷开发]] 中最流行的框架之一。

## 角色

- **Product Owner**：定义产品待办列表
- **Scrum Master**：确保流程顺畅
- **开发团队**：自组织、跨职能

## 事件

1. Sprint Planning
2. Daily Standup
3. Sprint Review
4. Sprint Retrospective

## 工件

- Product Backlog
- Sprint Backlog
- Increment

## 与看板的结合

很多团队会结合 [[看板方法]] 使用 Scrumban 模式，在 Sprint 内使用看板可视化。

## 适用场景

适合需要固定节奏交付的 [[项目管理]] 场景。需要良好的 [[团队协作]] 基础。
