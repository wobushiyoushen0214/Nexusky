# Nexusky 使用指南

## 安装与启动

### 从源码运行

```bash
pnpm install
pnpm run dev
```

### 使用安装包

从 Releases 下载对应平台的安装包：
- Windows: `.exe` 安装包或便携版
- macOS: `.dmg`
- Linux: `.AppImage`

### 首次使用

启动后会看到欢迎页面，选择：
- **创建笔记空间** — 新建一个空文件夹作为笔记库
- **打开已有文件夹** — 选择一个包含 `.md` 文件的目录

笔记库就是一个普通文件夹，所有笔记以 Markdown 文件存储。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+S` | 保存当前文件 |
| `Ctrl+O` / `Ctrl+P` | 快速切换笔记（模糊搜索） |
| `Ctrl+W` | 关闭当前标签 |
| `Ctrl+Tab` | 切换到下一个标签 |
| `Ctrl+N` | 新建笔记 |
| `Ctrl+G` | 知识图谱（右面板） |
| `Ctrl+Shift+G` | 知识图谱（全屏） |
| `Ctrl+L` | AI 对话面板 |
| `Ctrl+,` | 设置 |
| `Ctrl+Shift+F` | 全文搜索 |
| `Ctrl+Shift+B` | 折叠/展开侧边栏 |
| `Ctrl+Shift+P` | 命令面板 |
| `F11` | 聚焦模式（隐藏所有 UI） |
| `Tab` | 接受 AI 补全 |
| `Esc` | 取消 AI 补全 |

## 编辑器

### 基本操作

编辑器支持标准 Markdown 语法，输入时自动转换：
- `# ` → 一级标题
- `- ` → 无序列表
- `1. ` → 有序列表
- `- [ ] ` → 任务列表
- `> ` → 引用
- `` ` `` → 行内代码
- `==文本==` → 高亮

### 工具栏

编辑器顶部工具栏提供：标题、加粗、斜体、删除线、代码、列表、引用、表格、任务列表、高亮、分割线、导出。

### 图片

- 直接粘贴截图，自动保存到 `vault/assets/` 目录
- 拖拽图片文件到编辑器

### 导出

工具栏右侧可导出为 PDF 或 HTML。

---

## 双向链接

在编辑器中输入 `[[笔记标题]]` 创建双向链接：
- 链接以紫色高亮显示
- 点击跳转到对应笔记
- 如果目标笔记不存在，点击后自动创建
- 编辑器底部显示「反向链接」— 所有引用当前笔记的笔记列表

---

## 知识图谱

按 `Ctrl+G` 打开知识图谱面板，可视化所有笔记之间的链接关系：
- 节点大小按链接数缩放
- 鼠标悬停高亮相邻节点
- 点击节点跳转到对应笔记
- 当前打开的笔记有特殊高亮
- `Ctrl+Shift+G` 切换为全屏图谱视图

## 知识空间

知识空间把笔记作为可拖拽卡片放到无限画布中，适合整理项目、主题和时间线。

- 切换「空间 / 属性 / 时间」图层，分别按来源状态、主标签或更新时间组织卡片
- 拖拽卡片调整位置，位置会按 vault 和图层保存
- 画布会显示 wikilink 连接线，并在节点密集时绕开卡片
- 分组标签会提示当前区域的来源、状态、标签或日期
- 虚线连接表示系统发现的隐式关联，可在审阅队列中切换查看
- 接受隐式关联后，会把目标笔记写入来源笔记的 `## Connections` 区块

## 阅读收件箱

阅读收件箱集中处理 Notion、Readwise 和 Pocket 导入材料。

- 按来源、状态、关键词和归档状态筛选，来源计数会跟随当前条件变化
- 使用「下一条待读」连续处理阅读队列
- 可打开原文、打开原笔记或把阅读笔记追加到当前材料
- 可从当前筛选结果生成摘要笔记，保留摘录和精确文件路径 wikilink
- 可把当前筛选结果或选中条目交给 AI 消化，用于判断优先级、关联笔记和下一步动作
- 支持单条状态写回，以及当前结果批量标记已读、归档或取消归档

## AI 功能

### 配置 AI 提供商

1. 按 `Ctrl+,` 打开设置
2. 切换到「AI 提供商」标签
3. 点击「+ 添加」
4. 填写配置：
   - **名称**：随意命名，如 "My OpenAI"
   - **类型**：OpenAI / Claude / 自定义
   - **API Key**：你的 API 密钥
   - **Base URL**：留空使用官方地址，或填写中转站地址
   - **模型**：如 `gpt-4o-mini`、`claude-sonnet-4-6`
5. 点击「保存」，然后点击「激活」

#### 中转站配置

如果使用第三方中转站或国产模型：
- 类型选「自定义」
- Base URL 填中转站地址（如 `https://api.your-proxy.com/v1`）
- API Key 填中转站提供的密钥
- 模型填中转站支持的模型名

### AI 对话 (Ctrl+L)

打开右侧 AI 面板直接提问。Nexusky 会基于已索引的笔记内容进行本地检索，并在需要时用 AI 对候选片段重排序；回答会自动带上来源笔记。

未发送的输入会按当前笔记空间和 AI 会话自动保存草稿。切换会话或重新打开面板后，会恢复对应会话的草稿；发送后草稿会自动清除。

### AI 行内补全

写作时停顿约 2 秒，AI 会自动生成续写建议（灰色文字）：
- 按 `Tab` 接受建议
- 按 `Esc` 或继续输入取消

### AI 写作辅助

选中一段文本后，会出现浮动菜单，提供：
- 总结
- 扩展
- 改写
- 翻译
- 简化

### 语义索引

首次打开笔记库时会建立本地文本索引。语义搜索使用 FTS/TF-IDF 初筛，高置信度结果直接返回；模糊语义场景会调用当前 AI Provider 做轻量重排序。可在搜索面板手动重建索引。

## 云端同步

Nexusky 支持三种云端同步后端，在设置 > 云端同步中配置。

### 选择同步后端

打开设置 > 云端同步，顶部有三个按钮可切换：
- **Supabase** — 自建后端，完全可控
- **iCloud** — macOS 用户零配置
- **OneDrive** — 微软账户，跨平台

### Supabase

适合需要完全控制数据的用户。

1. 在 [supabase.com](https://supabase.com) 创建项目
2. 填入 Supabase URL 和 Key
3. 填入 Service Role Key（用于初始化和同步）
4. 点击「一键初始化」自动创建数据库表和 Storage bucket
5. 勾选「启用 Supabase」并保存

### iCloud Drive

macOS 用户最简单的选择，无需额外配置。

前提条件：
- 已登录 iCloud 账户
- 已启用 iCloud Drive

Nexusky 会自动检测 iCloud Drive 路径。如果检测不到，可以手动指定同步路径。同步的文件会自动同步到你的其他 Apple 设备。

### OneDrive

适合使用微软生态的用户。

配置步骤：
1. 在 [Azure Portal](https://portal.azure.com) 注册一个应用，获取 Client ID
   - 新建应用注册 → 重定向 URI 填 `http://localhost:23847/callback`
   - API 权限添加 `Files.ReadWrite.All` 和 `offline_access`
2. 在设置中填入 Client ID
3. 设置同步文件夹路径（默认 `/Nexusky`）
4. 点击「授权登录」，在弹出窗口中登录微软账户并授权

### 同步操作

配置好同步后端后，底部有两个操作按钮：

- **双向同步** — 推送本地变更到云端，同时拉取云端新文件到本地
- **从云端拉取** — 只从云端下载文件到本地，不上传

自动同步：每次保存文件（Ctrl+S）时，会自动推送到当前激活的云端后端。

## 搜索

### 快速切换 (Ctrl+O)

模糊搜索笔记标题，快速跳转。最近打开的文件排在前面。

### 全文搜索 (Ctrl+Shift+F)

搜索所有笔记的内容，显示匹配行和行号。

### 语义搜索

在 AI 对话中提问，或使用语义搜索功能，基于向量相似度找到语义相关的笔记。

### 命令面板 (Ctrl+Shift+P)

搜索并执行所有可用命令。

---

## 文件管理

- **文件树** — 侧边栏显示笔记库目录结构，支持展开/折叠
- **新建** — 侧边栏顶部按钮新建笔记或文件夹
- **右键菜单** — 重命名、删除、收藏
- **收藏** — 右键收藏的笔记显示在侧边栏顶部
- **每日笔记** — 侧边栏底部按钮，自动创建 `daily/YYYY-MM-DD.md`
- **模板** — 支持空白、会议记录、读书笔记、项目文档模板

## 外观

设置 > 外观中可切换内置主题、强调色、vault 级主题包和 CSS 片段。

主题包放在当前 vault 的 `.nexusky/themes/*.json`，刷新后即可在设置中启用。主题包只覆盖 Nexusky 已支持的 CSS 变量，例如：

```json
{
  "id": "ink",
  "name": "Ink",
  "colors": {
    "--bg-base": "#111111",
    "--bg-surface": "#1a1a1a",
    "--text-primary": "#f2f2f2",
    "--accent": "#7c6ef5"
  }
}
```

CSS 片段放在 `.nexusky/snippets/*.css`，会在主题包之后应用，适合做更细的界面微调。
